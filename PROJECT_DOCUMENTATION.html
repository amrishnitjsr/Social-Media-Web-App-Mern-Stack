🎯 Project Overview

A full-stack social media platform built using the MERN stack (MongoDB, Express.js, React.js, Node.js) that allows users to create profiles, share posts, interact with other users, and communicate in real-time.

Email/Password & Google OAuth with OTP verification

📝

❤️

👥

Instant messaging with typing indicators

🗄️ Database Models & Relationships

1. User Model

{
  email: String (required, unique),
  password: String (required, hashed),
  firstname: String (required),
  lastname: String (required),
  isAdmin: Boolean (default: false),
  profilePicture: String (image URL),
  coverPicture: String (image URL),
  about: String,
  livesin: String,
  worksAt: String,
  country: String,
  relationship: String,
  followers: [String] (Array of User IDs),
  following: [String] (Array of User IDs),
  googleId: String (unique, sparse),
  isGoogleUser: Boolean (default: false),
  online: Boolean (online status),
  timestamps: true (createdAt, updatedAt)
}

2. Post Model

{
  userId: String (required, references User),
  desc: String (post description/caption),
  likes: [String] (Array of User IDs who liked),
  image: String (image filename),
  comments: [
    {
      userId: String (references User),
      text: String (comment content),
      createdAt: Date (default: Date.now)
    }
  ],
  timestamps: true (createdAt, updatedAt)
}

3. InstaChat Model

{
  members: [String] (Array of User IDs, required),
  isGroup: Boolean (default: false),
  groupName: String (for group chats),
  groupAvatar: String (group image),
  lastMessage: String,
  updatedAt: Date (default: Date.now),
  blockedUsers: [String] (Array of blocked User IDs)
}

4. InstaMessage Model

{
  chatId: String (references InstaChat, required),
  sender: String (User ID, required),
  text: String (message content),
  image: String (image filename),
  status: String (enum: ['sent', 'delivered', 'read']),
  createdAt: Date (default: Date.now)
}

Key Controllers

Authentication Controller

Functions:
  • sendOtp - Sends OTP to Gmail for verification
  • verifyOtp - Verifies the OTP code
  • registerUser - Creates new user account
  • loginUser - Authenticates user and returns JWT

User Controller

Functions:
  • getUser - Retrieve user by ID
  • getAllUsers - Get all users (for suggestions)
  • updateUser - Update user profile
  • deleteUser - Delete user account
  • followUser - Follow another user
  • unFollowUser - Unfollow a user

Post Controller

Functions:
  • createPost - Create a new post
  • getPost - Get single post by ID
  • updatePost - Update post content
  • deletePost - Delete a post
  • likePost - Like/unlike a post
  • getTimelinePosts - Get posts from followed users
  • addComment - Add comment to post

⚡ Real-time Features (Socket.IO)

Socket Events

Event Direction Description
user_online Client → Server Mark user as online
user_offline Client → Server Mark user as offline
user_status Server → Client Broadcast user status
join_room Client → Server Join chat room
send_message Client → Server Send message
receive_message Server → Client Receive message
typing Client → Server User typing
stop_typing Client → Server User stopped typing
call_request Client → Server Initiate audio/video call
call_end Client → Server End the call

Message Flow

User A Server User B │ │ │ ├─send_message──────────>│ │ │ │ │ │ ├─Save to MongoDB │ │ │ │ │ ├─receive_message───────>│ │<─message_status─────────┤ │ │ (delivered) │ │ │ │<────message_read────────┤ │<─message_status─────────┤ │ │ (read) │ │

Connection Setup

// Client Side
import { io } from 'socket.io-client';

const socket = io('http://localhost:4000', {
  transports: ['websocket'],
  reconnection: true
});

socket.emit('user_online', userId);

// Server Side
io.on('connection', (socket) => {
  socket.on('send_message', (data) => {
    // Save message to database
    io.to(data.roomId).emit('receive_message', data);
  });
});

Protected Routes

// Middleware
const authMiddleware = async (req, res, next) => {
  const token = req.headers.authorization?.split(" ")[1];
  
  if (!token) {
    return res.status(401).json({ message: "No token" });
  }
  
  const decoded = jwt.verify(token, process.env.JWT_KEY);
  req.userId = decoded.id;
  next();
};

// Usage
router.get('/user/:id', authMiddleware, getUser);
